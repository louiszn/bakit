# Manual installation

Bakit is published on NPM and can be installed into your project manually.

## Requirements

- [Node.js](https://nodejs.org) v22 or newer.
- [Discord.js](https://discord.js.org) v14.

## Installing package

Install bakit and discord.js with your preferred package manager:

<Tabs>
	<TabItem value="npm" label="npm" default>
		```bash 
		npm install bakit discord.js
		```
	</TabItem>

    <TabItem value="pnpm" label="pnpm">
    	```bash
    	pnpm add bakit discord.js
    	```
    </TabItem>

    <TabItem value="yarn" label="yarn">
    	```bash
    	yarn add bakit discord.js
    	```
    </TabItem>

</Tabs>

<Admonition type="info" title="NOTE">
	Bakit is **TypeScript + ESM only**. Make sure you have a TypeScript project set up before using
	it. See **[Setting up project](#setting-up-project)**
</Admonition>

## Setting up project

### Installing TypeScript

Same as installing bakit, you can also install TypeScript with your package manager for your project:

<Tabs>
	<TabItem value="npm" label="npm" default>
		```bash 
		npm install --save-dev typescript @types/node
		```
	</TabItem>

    <TabItem value="pnpm" label="pnpm">
    	```bash
    	pnpm add --save-dev typescript @types/node
    	```
    </TabItem>

    <TabItem value="yarn" label="yarn">
    	```bash
    	yarn add --save-dev typescript @types/node
    	```
    </TabItem>

</Tabs>

### Configuring TypeScript

Configuring TypeScript correctly is an important step that determines your development experience. By default, you have to create `tsconfig.json` manually, or use `tsc --init` to generate an example config.

For using Bakit, you will need these options for your `tsconfig.json`:

- `target` as `es2024` - includes all features from Node.js v22.
- `module` as `nodenext` - controls ESM module correctly for Node.js environment.
- `experimentalDecorators` as `true` - get the latest decorator stage feature.
- `emitDecoratorMetadata` as `true` - get the metadata feature for decorator.

Setting `experimentalDecorators` and `emitDecoratorMetadata` is required in order to use bakit due to heavily decorator based API.

For example:

<Tabs>
	<TabItem value="code" label="tsconfig.json">
	
	```json
	{
		"compilerOptions": {
			"target": "es2024",
			"module": "nodenext",
			"strict": true,
			"skipLibCheck": true,
			"forceConsistentCasingInFileNames": true,
			"isolatedModules": true,
			"outDir": "dist",

    		// important!
    		"experimentalDecorators": true,
    		"emitDecoratorMetadata": true
    	},
    	"exclude": ["node_modules"]
    }
    ```

    </TabItem>

</Tabs>

<Admonition type="info" title="NOTE">
	This example is not always perfect for your use cases. It's recommended to have a look on official
	[TypeScript documentation](https://www.typescriptlang.org/tsconfig/).
</Admonition>

### Using ESM

Bakit doesn't and will never support CommonJS to reduce code complexity and package size, so using ESM is required.
To make your project use ESM, simply add or set `type` field to `module` in your `package.json`.

For example:

<Tabs>
	<TabItem value="code" label="package.json">
	
	```json
	{
		name: "example-bot",
		description: "Bakit is amazing!",
		type: "module",
		...
	}
	```

    </TabItem>

</Tabs>

If your project is currently in CommonJS, this [article](https://deno.com/blog/convert-cjs-to-esm) may help you a lot in migrating.
